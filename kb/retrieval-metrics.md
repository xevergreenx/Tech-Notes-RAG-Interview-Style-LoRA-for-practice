# Retrieval Metrics — конспект

## 1. Что такое Retrieval Metrics

**Retrieval Metrics** — это показатели качества поиска релевантных документов в системах RAG, semantic search или QA.

Цель:

* понять, насколько найденные документы полезны для ответа
* оценить эффективность embeddings + vector DB

---

## 2. Основные метрики

### 2.1 Recall@k

* Доля релевантных документов среди top-k найденных
* Формула:

```
Recall@k = (# релевантных документов в top-k) / (# релевантных документов в базе)
```

* Используется для оценки полноты поиска

### 2.2 Precision@k

* Доля релевантных документов среди top-k найденных

```
Precision@k = (# релевантных документов в top-k) / k
```

* Оценивает точность поиска

### 2.3 MRR (Mean Reciprocal Rank)

* Среднее обратное рангу первого релевантного документа

```
MRR = (1/N) * Σ (1 / rank_i)
```

* Чем выше — тем быстрее релевантный документ находится

### 2.4 nDCG (Normalized Discounted Cumulative Gain)

* Учитывает позиции и релевантность нескольких документов
* Формула:

```
DCG = Σ (2^rel_i - 1) / log2(i+1)
nDCG = DCG / IDCG
```

* Используется, если релевантность документов имеет градации (1–3)

---

## 3. Дополнительные метрики

* **Hit@k** — есть ли хотя бы один релевантный документ в top-k
* **Cosine similarity score** — среднее значение схожести найденных документов
* **Latency / Speed** — время поиска

---

## 4. Применение в RAG

1. Формируем эмбеддинги документов и сохраняем в vector DB
2. Запрос → embedding → top-k поиск
3. Сравниваем с ground-truth релевантными документами
4. Считаем Recall@k, MRR, nDCG

---

## 5. Практические советы

* k = 5–10 для оценки retrieval
* Для QA важен **Recall**, чтобы релевантный документ точно был найден
* Для summarization / RAG важен **nDCG**, чтобы лучшие документы были выше
* Проверяй Latency, чтобы не тормозило генерацию LLM

---

## 6. Интуитивно

* Recall = «Нашёл ли модель то, что нужно?»
* Precision = «Сколько найденного действительно полезно?»
* MRR = «Насколько быстро нашёл первый правильный документ?»
* nDCG = «Сильно ли ранжирование соответствует важности?»

---

## 7. Краткое резюме

Retrieval Metrics — ключ к оценке качества RAG.

* Без метрик нельзя понять, **работает ли поиск**
* Они позволяют выбирать эмбеддер, vector DB и оптимизировать k для топовых результатов
* Правильные метрики — основа точного, полезного и быстрого RAG
