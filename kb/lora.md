# LoRA — конспект

## 1. Что такое LoRA

**LoRA (Low-Rank Adaptation)** — метод дообучения больших языковых моделей, который позволяет адаптировать модель к новой задаче, изменяя только небольшие матрицы внутри трансформера.

Цель:

* снизить потребление GPU
* ускорить обучение
* сохранить исходные знания модели

---

## 2. Идея LoRA

* Веса модели W ∈ ℝ^(d×k)
* LoRA добавляет низкоранговые матрицы A и B:

```
W' = W + α * A * B
```

* α — коэффициент масштабирования
* Только A и B обучаются, W остаётся замороженным

Плюсы:

* экономия памяти
* меньше данных для обучения
* легко интегрируется в любую LLM

---

## 3. Применение

### 3.1 Подготовка данных

* Формат instruction → response (как для SFT)
* Чанки ≤ max_length модели
* Очистка шума

### 3.2 Настройка тренинга

* Библиотеки: [PEFT](https://github.com/huggingface/peft), Transformers
* Параметры:

  * r (rank) = 8–32
  * α = 16
  * lr = 2e-4
  * batch_size = 1–8
* Замораживаем все веса модели, обучаем только LoRA матрицы

---

### 3.3 Пример кода

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen2.5-7B-Instruct"
model = AutoModelForCausalLM.from_pretrained(model_name, device_map="auto")
tokenizer = AutoTokenizer.from_pretrained(model_name)

config = LoraConfig(
    r=16,
    lora_alpha=16,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)
model = get_peft_model(model, config)
```

---

## 4. Особенности LoRA

* Меняет **только часть модели** → быстро и экономно
* Совместим с FP16 / BF16
* Можно хранить отдельно: вес LoRA ~ десятки МБ вместо сотен ГБ

---

## 5. LoRA + RAG

* LoRA полезен для стилизации ответов или instruction tuning
* RAG обеспечивает фактические знания
* Комбинация:

  * RAG = факты
  * LoRA = стиль / тон / специфическая область

---

## 6. Преимущества

✅ Малые требования к GPU
✅ Быстрое обучение
✅ Можно легко обновлять несколько LoRA для разных задач
✅ Минимальный риск «забывания» базовых знаний модели

---

## 7. Ограничения

❌ Не обучает новые знания — только адаптация
❌ Rank слишком маленький → модель может плохо подстроиться
❌ Иногда нужен SFT перед LoRA, чтобы улучшить instruction-following

---

## 8. Практические советы

* Начинай с r=8–16
* target_modules: q_proj, v_proj (в трансформере decoder)
* Используй LoRA для SFT, а не для factual knowledge
* Сохраняй LoRA отдельно, можно накладывать на разные базовые модели

---

## 9. Интуитивно

LoRA = «надстройка» над моделью:

* базовая LLM остаётся нетронутой
* LoRA корректирует ответы под стиль, тон, формат

---

# Краткое резюме

LoRA — эффективный способ адаптировать LLM под задачи:

* быстро
* экономно
* с минимальными ресурсами
* идеально в связке с RAG для instruction tuning
