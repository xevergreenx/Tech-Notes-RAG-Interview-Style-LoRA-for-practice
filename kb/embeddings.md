# Embeddings — конспект

## 1. Что такое embeddings

**Embeddings** — это векторы чисел, которые представляют объекты (текст, токены, изображения) в пространстве признаков.

Главная идея:
похожие объекты имеют близкие векторы.

Пример:

```
"кошка" ≈ "кот"
"машина" ≈ "автомобиль"
```

---

## 2. Типы embeddings

### 2.1 Token embeddings

* Каждый токен в тексте получает вектор
* Используется внутри LLM
* Не предназначен для поиска или similarity напрямую

### 2.2 Sentence / Text embeddings

* Представляют целую фразу или документ
* Используются для:

  * semantic search
  * clustering
  * RAG

---

## 3. Как создаются embeddings

### 3.1 Модель

* Обычно трансформеры (BERT, RoBERTa, E5)
* Выходной вектор фиксированной размерности (например 768 или 1024)

### 3.2 Математика

1. Токены → embedding layer → hidden states
2. Pooling (mean / CLS) → итоговый вектор
3. Нормализация (иногда L2)

---

## 4. Свойства embeddings

* Фиксированная размерность
* Косинусное сходство отражает семантическую близость
* Можно складывать, усреднять, сохранять

Пример:

```python
similarity = cosine(vec1, vec2)
```

---

## 5. Использование в RAG

1. Тексты документов → embeddings → vector DB
2. Вопрос → embeddings → поиск top-k ближайших
3. Контекст → LLM → генерация ответа

---

## 6. Почему embeddings лучше, чем токены для поиска

* Семантика, а не просто совпадение слов
* Учитывают смысл всей фразы
* Позволяют находить релевантные документы даже без точного совпадения слов

---

## 7. Примеры моделей для embeddings

* **intfloat/multilingual-e5-base** — универсальный multilingual embedder
* **all-MiniLM-L6-v2** — быстрый, лёгкий
* **LaBSE** — для многоязычных задач
* **OpenAI text-embedding-3-large** — облачные embeddings

---

## 8. Метрики оценки

* Cosine similarity
* Euclidean distance (L2)
* Recall@k / Precision@k (для retrieval)
* Clustering metrics

---

## 9. Интуитивно

Embeddings = карта смыслов:

* похожие тексты рядом
* разные тексты далеко
* LLM смотрит на embeddings → понимает, что взять в контекст

---

## 10. Практическая заметка

* Нормализуй векторы для cosine similarity
* Разбивай большие документы на чанки
* Используй качественные модели embeddings для RAG

---

# Краткое резюме

Embeddings — это числовое представление смысла текста, которое позволяет:

* находить похожие документы
* формировать контекст для LLM
* ускорять поиск и улучшать точность ответов

В RAG embeddings — **ядро поиска**, без них LLM будет только гадать.
