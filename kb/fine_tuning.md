# Fine-Tuning — конспект

## 1. Что такое Fine-Tuning

**Fine-Tuning** — это дообучение уже предобученной LLM на специфическом датасете для конкретной задачи или стиля.

Цель:

* улучшить стиль или тон
* адаптировать к узкой предметной области
* научить модели выполнять определённые инструкции

---

## 2. Типы Fine-Tuning

### 2.1 Full Fine-Tuning

* Обучаются **все веса модели**
* Плюсы: максимальная адаптация
* Минусы: дорого по ресурсам, риск «забыть» базовые знания

### 2.2 LoRA / QLoRA

* Дообучаются только **небольшие адаптивные матрицы**
* Плюсы: дешево, быстро, меньше VRAM
* Минусы: меньше адаптации, чем full fine-tune

### 2.3 Instruction Tuning (SFT)

* Модель дообучают на парах **prompt → ответ**
* Цель: сделать модель **instruction-following**

---

## 3. Шаги Fine-Tuning

### 3.1 Подготовка данных

* Формат: instruction → response
* Примеры:

```json
{
  "prompt": "Объясни статью 105 УК РФ",
  "response": "Статья 105 УК РФ описывает убийство..."
}
```

* Очистка:

  * короткие / пустые ответы
  * шумные данные

### 3.2 Токенизация

* Используется тот же tokenizer, что у модели
* max_length = длина контекста + генерация

### 3.3 Настройка тренера

* Библиотеки: Transformers, PEFT, TRL
* Параметры:

  * learning_rate: 2e-4
  * batch_size: 1–8
  * num_train_epochs: 1–3
  * fp16 / bf16

### 3.4 Обучение

* Trainer / SFTTrainer / LoRATrainer
* Отслеживание loss и возможной деградации

---

## 4. Советы при Fine-Tuning

* Фильтруй токсичные данные
* Используй small batch для стабильности
* Добавь системный prompt для consistency
* Сохраняй модель после каждой эпохи

---

## 5. Когда делать Fine-Tuning

✅ Нужен стиль ответа
✅ Узкая область знаний
✅ Формат (например FAQ, код, юридические ответы)

❌ Не нужен для базовых знаний или RAG
❌ Не нужен, если знания часто обновляются

---

## 6. Интеграция с RAG

* RAG отвечает за фактические знания
* Fine-Tuning помогает **форматировать ответы и стиль**
* Хорошо комбинируется: RAG + SFT → точность + стиль

---

## 7. Проблемы

❌ Переобучение (overfitting)
❌ Забывание базовых знаний (catastrophic forgetting)
❌ Большие затраты GPU
❌ Некорректный prompt → некорректный ответ

---

## 8. Метрики

* Loss (cross-entropy)
* Accuracy на QA / instruction
* Human evaluation (style, helpfulness)
* Perplexity

---

## 9. Интуитивно

Fine-Tuning = **докормить LLM нужной пищей**, чтобы она говорила:

* правильно
* в нужном стиле
* по нужной теме

---

# Краткое резюме

Fine-Tuning — мощный инструмент, но требует аккуратности:
LoRA / QLoRA позволяют экономить ресурсы,
SFT формирует полезный стиль,
а RAG решает проблему актуальных знаний.
