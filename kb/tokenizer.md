# Tokenizer — конспект

## 1. Что такое Tokenizer

**Tokenizer** — это инструмент, который разбивает текст на токены и преобразует их в числа для подачи в LLM или другие модели NLP.

Цель: перевод текста в форму, понятную модели.

---

## 2. Почему нужен токенизатор

* Модели работают с числами, а не с текстом
* Позволяет учесть subwords, слова и спецсимволы
* Оптимизирует размер словаря и эффективность обучения

---

## 3. Типы токенизации

### 3.1 Word-level

* Каждый токен = отдельное слово
* Проблемы:

  * большой словарь
  * неизвестные слова → <UNK>

### 3.2 Subword-level (BPE / WordPiece / SentencePiece)

* Токен = часть слова
* Пример:

```
"машинное обучение" → ["машин", "ное", "обучение"]
```

* Решает проблему OOV (out-of-vocabulary)
* Используется в большинстве современных LLM

### 3.3 Character-level

* Каждый символ = токен
* Преимущества:

  * нет OOV
  * точная сегментация
* Минусы:

  * длинные последовательности → больше токенов

---

## 4. Процесс токенизации

1. Text → Tokenizer
2. Tokenizer → Sequence of token ids
3. Добавление special tokens (BOS, EOS, PAD, MASK)
4. Padding / Truncation для одинаковой длины последовательностей

Пример (Python + Hugging Face):

```python
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-7B-Instruct")
tokens = tokenizer("Привет, как дела?", return_tensors="pt")
print(tokens)
```

---

## 5. Де-токенизация

* Преобразование токенов обратно в текст
* Используется для генерации ответа LLM

Пример:

```python
decoded = tokenizer.decode(tokens["input_ids"][0], skip_special_tokens=True)
```

---

## 6. Специальные токены

| Token  | Назначение                            |
| ------ | ------------------------------------- |
| [PAD]  | Padding для одинаковой длины          |
| [BOS]  | Начало текста (beginning of sequence) |
| [EOS]  | Конец текста (end of sequence)        |
| [MASK] | Маскирование для MLM                  |

---

## 7. Почему токенизация важна для LLM и RAG

* Контекстное окно модели ограничено в токенах → правильная токенизация = больше текста в окне
* Embeddings строятся на токенах → качественная токенизация улучшает semantic search
* LoRA / SFT / RAG требуют согласованной токенизации для обучения и inference

---

## 8. Практические советы

* Для русских текстов использовать multilingual или русскоязычные токенизаторы
* Всегда проверять special tokens при генерации
* Разбивать длинные документы на чанки по токенам, а не по словам

---

## 9. Интуитивно

Tokenizer = «мост» между текстом и моделью:

* Текст → числа → LLM
* Числа → текст → человек видит результат

---

# Краткое резюме

Tokenizer — обязательный элемент NLP и LLM:

* разбивает текст на токены (слова / subwords / символы)
* преобразует в числовые id
* добавляет специальные токены (BOS, EOS, PAD, MASK)
* влияет на эффективность обучения, генерации и RAG
