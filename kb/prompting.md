# Prompting — конспект

## 1. Что такое Prompting

**Prompting** — это искусство и наука составления входных инструкций (prompts) для LLM или RAG, чтобы получить нужный ответ.

Цель:

* направить модель
* контролировать стиль, тон и формат
* повысить factual accuracy в RAG

---

## 2. Основные типы промптов

### 2.1 Instruction-based

* Прямое указание модели, что делать
* Пример:

```
Объясни статью 158 УК РФ простыми словами.
```

### 2.2 Few-shot

* Даем примеры input → output для демонстрации формата
* Пример:

```
Q: Что такое кража?
A: Тайное хищение чужого имущества.

Q: Что такое мошенничество?
A: Обман с целью получить выгоду.

Q: Что такое убийство?
A: ...
```

### 2.3 Zero-shot

* Только инструкция без примеров
* Пример:

```
Составь список правонарушений, упомянутых в УК РФ.
```

### 2.4 Chain-of-Thought (CoT)

* Просим модель рассуждать пошагово
* Увеличивает точность сложных задач
* Пример:

```
Объясни пошагово, какие действия подпадают под состав преступления по статье 158 УК РФ.
```

---

## 3. Prompting для RAG

1. Формируем retrieved context (чанки из vector DB)
2. Включаем контекст в prompt:

```
Контекст:
<retrieved chunks>

Вопрос:
<user query>

Ответь подробно, опираясь на контекст.
```

3. Можно добавлять системные инструкции:

```
- Используй формальный стиль
- Давай только фактическую информацию
- Не придумывай детали
```

---

## 4. Параметры prompt engineering

* **Temperature**: 0–0.7 для factual tasks
* **Top-p / Top-k**: ограничение выбора токенов → меньше "галлюцинаций"
* **Max tokens**: чтобы не обрезать ответ
* **Stop tokens**: завершение генерации в нужной точке

---

## 5. Примеры промптов

### 5.1 Для LLM

```
Составь краткий конспект главы 1 УК РФ.
```

### 5.2 Для RAG

```
Контекст:
[Статья 105: Убийство, умышленное причинение смерти]
[Статья 107: Убийство по неосторожности]

Вопрос:
Объясни разницу между статьями 105 и 107 УК РФ простыми словами.
```

---

## 6. Практические советы

* Всегда включай **контекст** для factual запросов
* Для style / tone → используй инструкции напрямую
* Few-shot + CoT → увеличивает точность
* Не перегружай prompt → слишком длинные → risk of truncation

---

## 7. Интуитивно

Prompt = «команда для модели»:

* чем точнее команда → тем точнее ответ
* для RAG добавляем «контекст» → чтобы модель не выдумывала
* можно «учить» стиль без fine-tuning через правильный prompt

---

# Краткое резюме

Prompting — ключевой навык для работы с LLM и RAG:

* Instruction / Zero-shot / Few-shot / CoT
* Системные инструкции + retrieved context
* Настройка генерации через temperature, top-p, max_tokens
* Хороший prompt = точный, понятный, релевантный ответ
